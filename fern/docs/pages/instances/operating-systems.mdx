## Operating systems

When you [create an instance](/api-reference/instances/create), you can specify an operating system (OS) template. To do so, provide one of the available template names as the value for `operating_system_label`.

<Note>If you do not provide a value for `operating_system_label` when creating an instance, `"ubuntu_20_04_lts"` is used as the default.</Note>

### Get the list of available OS templates

For the most up-to-date list, call the [`List available OS templates`](/api-reference/templates/list) endpoint.

Example request:
<EndpointRequestSnippet endpoint="GET /list_available_os_templates" />

Make sure to replace the `<api_key>` with your own [API key](/get-started/api-overview#authentication)

The endpoint returns a list of operating systems, as shown in this example response:
<EndpointResponseSnippet 
  endpoint="GET /list_available_os_templates"
/>

Use one of the returned values for the `label` property as the value for `operating_system_label` when you create an instance.

### List of available OS templates

| name                       | description                                               | label                    |
|----------------------------|-----------------------------------------------------------|--------------------------|
| Ubuntu 22.04 LTS           | Ubuntu 22.04 LTS plain                                    | ubuntu_22_04_lts         |
| Ubuntu 20.04 LTS           | Ubuntu 20.04 LTS plain                                    | ubuntu_20_04_lts         |
| Ubuntu 20.04 LTS (Nvidia)  | Ubuntu 20.04 LTS with Nvidia/CUDA 12 drivers preinstalled | ubuntu_20_04_lts_nvidia  |
| Ubuntu 22.04 LTS (Nvidia)  | Ubuntu 22.04 LTS with Nvidia/CUDA 12 drivers preinstalled | ubuntu_22_04_lts_nvidia  |
